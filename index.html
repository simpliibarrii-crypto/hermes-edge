SYSTEM PRINCIPLES

Local first does not mean claims first.

Hermes is designed around measurable execution contracts instead of vague “runs on edge” language.

01 / TOOL FIRST

Use the smallest reliable executor.

Ordinary calculations and registered deterministic actions should not consume model inference.

02 / DEVICE AWARE

Route around real hardware.

Memory, runtime availability, backend support, and fallback behaviour shape every route decision.

03 / AUDITABLE

Keep the route visible.

Model profile, backend, fallback, and benchmark context remain inspectable by operators.

BENCHMARK CONTRACT

Performance needs coordinates.

A number without its device, backend, model, memory, and thermal state is marketing fog.

Required metadata

Every public performance result should identify the execution environment.

Minimal local path

Install the lightweight core, run policy tests, then add the runtime extra only where supported.

git clone https://github.com/simpliibarrii-crypto/hermes-edge.git
cd hermes-edge
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
pytest tests/test_edge_policy.py \
  tests/test_agent_edge_routing.py \
  tests/test_litert_backend.py -q
